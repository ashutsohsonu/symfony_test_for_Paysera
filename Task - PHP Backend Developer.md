# Homework assignment

## Task
Create a secure API for transferring funds between accounts. This system will demonstrate your ability to develop scalable and reliable financial applications that can handle high loads.

*(This isn’t a full payment system — just one or a couple clean, reliable endpoints. We’re evaluating your ability to ship robust, production-ready components — not just working code.)*


## Core Requirements

**Technology Stack:** Use the latest PHP with Symfony and MySQL + Redis for appropriate persistence and caching layers for handling high-load scenarios.

**Key Challenges:**
- Implement fund transfers between accounts
- Ensure system reliability under load
- Maintain transaction integrity

**Quality Standards:**
- Follow modern PHP development practices and code organization
- Include comprehensive test coverage with integration tests
- Implement proper error handling and logging
- Create clear documentation for setup and usage


## Evaluation Criteria

We'll assess your submission based on:
- **Technical Excellence:** Code quality, architecture decisions, and best practices
- **Problem Solving:** Your approach to technical challenges and edge cases
- **Modern Practices:** Use of contemporary PHP/Symfony patterns and tooling
- **Attention to Detail:** Data persistence, validation, security considerations
- **Professional Readiness:** Documentation, testing, coding style


## Creative Freedom

**AI-Assisted Development:** We encourage you to use AI tools (such as GitHub Copilot, ChatGPT, or similar) during development — **leveraging AI effectively is an important part of our daily workflow.**  
However, please ensure you fully understand and can explain any code generated with AI, as your submission reflects your professional judgment and technical ownership.

**Architecture Decisions:** The implementation approach, API, and data storage structure are entirely up to you. Show us your decision-making process and attention to detail.

**Beyond Minimums:** This is your opportunity to demonstrate expertise beyond the core requirements.



***It's perfectly acceptable if you don't implement every feature you envision. In such a case, document your thought process and outline what additional improvements you would make. This demonstrates your planning abilities and shows how you would make the solution more robust in a real-world scenario.***


### How to Submit

Since this task involves a Symfony application, please:

1. **Create a public GitHub repository** with your solution.
2. **Make sure it includes:**
   - All necessary code (`src/`, `config/`, `public/`, etc.)
   - `composer.json`, `composer.lock`,  docker files, etc.
   - A short **README.md** with:
     - How to install and run the app (e.g., `composer install`, `php bin/console ...`)
     - **Approximate time spent** (e.g., "Time spent: ~2 hours")
     - **AI tools and prompts used**
3. **Send the link to your repository** to the interviewer.
